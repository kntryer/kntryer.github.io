2016 年度总结

发表于 2016-12-28 | 分类于 随笔 ， 年度总结 | 阅读次数:

  去年11月份，在工作室的指导老师的介绍下，我们工作室几个人承接了一个APP的外包。没有任何外包经验的我们摸爬滚打，常常开几个钟的会来讨论功能需求，讨论进度。

  学习新的东西，时间总是很短。一月份很快就来临，因为经验的问题，我们的开发周期延长到这个月。半吊子 Android 的我在学东西很慢，很多东西都不理解，遇到 bug 的时候，一天很快就过去了。没有做过的东西都会觉得很难，但坚持下去一定会有收获。

  中旬的时候，忍受不了老妈的催促，我回家了。因为那时候奶奶的身体有恙，我想回去看看。在家的每天都会抽空看看奶奶还有爷爷，他们蛮开心的，我想老年时最盼的是儿孙的关心与陪伴吧。

  不得不说，我还挺喜欢瞎搞的。那时家里没有拉网线，搞开发变得很艰难。刚好家里有个路由器，就把邻居的 WiFi 覆盖到咱家来，虽然网速不稳定，但至少能查查资料，看看博客。我对编程这么感兴趣，可能很大部分来自于瞎搞，嘿嘿。

  二月份在忙碌的气氛中悄悄到来，整个埠头都忙着搞卫生、买年货、贴对联等等，脸上布满幸福的笑容。忙碌是春节的气氛吧，突然这样想。每年除夕的那晚我都没睡，静静的看着空白的天花板，放空自己，让思绪自动流转。看起来毫无意义，但很舒适。

  有相聚必有离别。在离开的那天，看着奶奶浑浊的双眼，有点不舍，我握住奶奶的手，对她说：“我过完这学期就回来看你，很快”。来到了九月，没想到这一别就是永远。

  三四月在忙一个大学生软件设计比赛，和舍友一起进行开发，那时我开始学习php接口开发，结果看了一周的php就开始打代码了。

设计模式之单例模式

发表于 2016-04-26 | 分类于 技术笔记 ， Java | 阅读次数:

  单例模式是一种比较常用的设计模式，提供了对唯一实例的受控访问，由于在系统内存中只存在一个对象，因此可以节约系统资源，对于一些需要频繁创建和销毁的对象，单例模式无疑可以提高系统的性能。

  但是，由于单利模式中没有抽象层，因此单例类的扩展有很大的困难。滥用单例将带来一些负面问题，如为了节省资源将数据库连接池对象设计为的单例类，可能会导致共享连接池对象的程序过多而出现连接池溢出。
